**Key Features:**

- **Designed to remove friction:** Real-time card validation with built-in error messaging.
- **Mobile-ready:** Fully responsive design.
- **International:** Supports 13 languages.
- **Multiple payment methods:** Supports various [payments methods](https://monei.com/blog/multiple-payment-options/).

- **Fraud and compliance:** Simplified PCI compliance and SCA-ready.

You can customize the appearance in your [MONEI Dashboard → Settings → Branding](https://dashboard.monei.com/settings/branding).

## Integration Flow

The integration involves redirecting your customer to MONEI's secure page and handling their return to your site.

### 1. Create a Payment Resource on Your Server

First, make a server-side API call to create a new payment object. This registers the payment intent with MONEI.

**Required Parameters:**

- **amount** `positive integer`: Amount in the smallest currency unit (e.g., 110 for €1.10).
- **currency** `string`: Three-letter [ISO currency code](https://en.wikipedia.org/wiki/ISO_4217) (e.g., `EUR`).
- **orderId** `string`: Your unique order identifier for reconciliation.
- **completeUrl** `string`: The URL the customer is redirected to **after** attempting payment (successful or failed). They return here after interacting with the MONEI page or 3D Secure.
- **callbackUrl** `string`: Your server endpoint URL where MONEI sends an **asynchronous webhook notification** about the final payment status. This is crucial for confirming payment status reliably.
- **cancelUrl** `string`: The URL the customer is redirected to if they explicitly click the **cancel** or **"Back to shop"** link on the MONEI payment page before completing the payment.

Check all available [request parameters](/apis/rest/payments-create/).

### 2. Redirect the Customer to MONEI

The API response from Step 1 contains a `nextAction` object with a `redirectUrl`. You **must** redirect your customer's browser to this URL.

```json
{
  "id": "af6029f80f5fc73a8ad2753eea0b1be0", // MONEI Payment ID
  "amount": 110,
  // ... other fields ...
  "status": "PENDING", // Initial status
  "nextAction": {
    "type": "CONFIRM",
    "mustRedirect": true, // Indicates redirection is needed
    "redirectUrl": "https://secure.monei.com/payments/af6029f80f5fc73a8ad2753eea0b1be0" // <-- REDIRECT HERE
  },
  "createdAt": 1594215339
}
```

This `redirectUrl` leads the customer to the secure, MONEI-hosted payment page.

Instead of redirecting, you can use [`monei.js`](/monei-js/reference/) to present the payment options within a modal on your own site. See the [Payment Modal Guide](/integrations/use-payment-modal.mdx) for details.

### 3. Customer Interacts with MONEI Page

On the MONEI page, the customer selects a payment method, enters their details, and completes any necessary authentication steps (like 3D Secure).

### 4. Customer is Redirected Back to Your Site

After the customer completes (or cancels) the process on the MONEI page, they are redirected back to one of the URLs you provided in Step 1:

- **Success/Failure (`completeUrl`):** If the payment is attempted (successfully or not), or if 3D Secure is completed, the customer is redirected to your `completeUrl`. The MONEI `paymentId` is appended as a query parameter (e.g., `https://example.com/checkout/complete?paymentId=af6029f80f5fc73a8ad2753eea0b1be0`).
  - **Action:** Use the `paymentId` from the URL to call the [Get Payment API](/apis/rest/payments-get/) to fetch the final payment status and display an appropriate confirmation or error message to the customer. **Do not rely solely on the redirect to confirm success.**
- **Cancellation (`cancelUrl`):** If the customer clicks the **"Back to \{shop.name}"** link or a cancel button _before_ attempting payment, they are redirected to your `cancelUrl`.
  - **Action:** Typically, redirect the user back to their cart or the previous step in your checkout flow.

### 5. Receive Asynchronous Confirmation via Webhook

Independently of the customer's browser redirection, MONEI sends an **asynchronous** HTTP POST request to your `callbackUrl` once the payment reaches a final state (e.g., `SUCCEEDED`, `FAILED`).

- **Payload:** The request body contains the full [Payment object](/apis/rest/schemas/payment/) in JSON format.
- **Verification:** The request includes a `
